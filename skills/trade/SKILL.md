# Trade

Use this skill for four workflows:

1. `正式选股`：基于本仓库交易规则，对候选股逐只打分，给出当日尾盘可执行建议。
2. `交易日志`：用户说出买入、卖出、减仓、撤单后，增量追加记录。
3. `模拟选股`：忽略时间限制，按最新完整日线范围做盘后模拟筛选。
4. `卖出分析`：买入后任意时刻，结合持仓记录、当前结构和卖出规则，给出持有、减仓或卖出建议。

4. [tools/trade_risk_tool.html](../../tools/trade_risk_tool.html) 仅在需要解释仓位和风险时读取

## 工作边界

这个 skill 不自带实时行情源。若仓库内没有最新候选池、行情快照或用户未提供导出数据，必须明确说明数据不足，并要求用户提供以下任一输入：

1. i问财结果导出
2. 通达信/同花顺自选或公式筛选结果
3. 手工给出的股票名单和关键价格

不要假装拥有实时行情。

若做 `卖出分析`，优先读取 `records/trade_journal.jsonl` 中对应股票的买入记录；若日志不存在或信息不足，必须明确说明分析退化为“仅基于当前描述的卖出讨论”，置信度下降。

## 空输入时怎么响应

如果用户直接调用 `trade` skill，但没有提供任何内容，先不要自行分析，先输出一段简短引导，告诉用户这个 skill 可以做什么，以及可以怎么问。

默认引导语应覆盖这几类场景：

1. `买入分析`
   你可以这样问：
   `现在 14:30 以后，帮我复核这批候选股，给出尾盘买入建议。`

2. `模拟选股`
   你可以这样问：
   `帮我做模拟选股，基于这批候选股按最新完整日线评分。`

3. `卖出分析`
   你可以这样问：
   `我持有 000001，买入价 12.36，帮我做卖出分析。`

4. `记录买入`
   你可以这样问：
   `我刚买入 000001，成交价 12.36，止损价 11.86，帮我记录交易日志。`

5. `记录卖出`
   你可以这样问：
   `我刚卖出 000001，卖出价 13.20，原因是到目标位，帮我记录交易日志。`

引导语要短，优先起到提示作用，不要展开成长篇说明。

## 模式规则

### 1. 正式选股

正式选股默认使用 `Asia/Shanghai` 本地时间判断。

1. 若当前时间早于 `14:30`，不输出“今日可买入推荐股”。
2. 若用户坚持在 `14:30` 前查看，只能输出 `观察名单`，并明确标注“未到尾盘，不构成买入建议”。
3. `14:30` 之后才允许给出 `尾盘推荐股`。
4. 尾盘推荐仍必须满足：周线多头、日线回调、尾盘未走坏、风险收益比合格、市场情绪不过热或不过冷。

#### T-1 候选池规则

允许用户在 `T-1` 交易日收盘后提供候选股，例如 `6月13日 15:00 后` 提供，准备在下一个交易日 `T` 的尾盘复核。

这里必须区分两层：

1. `T-1 盘后候选池`：只解决“明天哪些票值得盯”，属于观察池。
2. `T 日尾盘推荐`：只解决“今天 14:30 之后哪些票仍可执行买入”。

因此：

1. `T-1` 候选池可以用于 `T` 日正式选股的输入。
2. `T-1` 候选池不能直接等同于 `T` 日买入名单。
3. `T` 日 `14:30` 之后必须对候选股重新复核，不允许直接沿用 `T-1` 的买入结论。
4. 若 `T` 日已经出现追高、破位、板块转弱、市场情绪极值、收益风险比恶化，则必须剔除。

#### T 日复核要求

当用户拿 `T-1` 候选池来做 `T` 日尾盘决策时，至少复核：

1. 周线多头是否仍成立
2. 日线回调是否仍属于系统内位置
3. 当天是否已经从“回调可买”变成“拉高追价”
4. 当天是否跌破关键支撑且尾盘未收回
5. 板块和市场情绪是否支持出手
6. 计划买入价、止损价、目标价和收益风险比是否仍成立

若缺少 `T` 日尾盘状态，只能输出 `观察`，不能输出 `推荐`。

### 2. 模拟选股

当用户明确提到 `模拟选股` 时：

1. 忽略 `14:30` 时间限制。
2. 使用最新完整日线区间，不使用未收盘日线做结论。
3. 输出时必须标注 `模拟结果，不代表盘中可执行建议`。
4. 若输入是 `T-1` 盘后候选池，则可将其作为模拟复盘样本，但结论仍以最新完整日线为准，而不是沿用旧评分。

### 3. 交易日志

当用户提到已 `买入`、`卖出`、`减仓`、`清仓`、`撤单` 时：

1. 追加写入 `records/trade_journal.jsonl`。
2. 永远增量追加，不覆盖旧记录。
3. 若用户修正上一条记录，新增一条 `correction` 事件，并引用原时间或原股票代码。
4. 若关键信息缺失，先记录已知字段，再把缺失项放进 `missing_fields`。

### 4. 卖出分析

当用户说 `卖不卖`、`帮我看卖点`、`是否止盈`、`是否止损`、`是否减仓`、`卖出分析` 时，进入这个模式。

卖出分析优先级：

1. 先找该股票最近的买入记录
2. 再读取用户提供的当前价格、截图、候选数据或外部辅助信息
3. 最后按仓库卖出规则做结构分析

如果买入记录可用，至少回看：

1. 原买入日期
2. 原买入理由
3. 原计划买入价、止损价、目标价
4. 原评分和市场情绪
5. 原计划卖出条件

如果没有历史买入记录，也可以做卖出分析，但必须明确标注：

1. `未匹配到买入日志`
2. `无法验证是否偏离原计划`
3. 结论只基于当前结构，不基于完整交易闭环

## 评分框架

沿用仓库的 100 分制度，但把“尾盘买入”和“市场情绪”并入执行层：

1. 周线趋势 `25`
2. 日线回调质量 `20`
3. 尾盘触发质量 `15`
4. 板块强度 `10`
5. 收益风险比 `15`
6. 执行纪律与市场情绪 `15`

执行阈值：

1. `>= 80`：允许进入买入审批
2. `70-79`：仅观察或轻仓预案，不主动买入
3. `< 70`：不交易

旧评分只可用于排序观察优先级，不能直接作为下一交易日的执行评分。只要跨到新的交易日，尾盘执行分至少要重算以下三项：

1. 尾盘触发质量 `15`
2. 板块强度 `10`
3. 执行纪律与市场情绪 `15`

若 `T` 日价格结构明显变化，日线回调质量和收益风险比也必须同步重算。

卖出分析不再使用“是否可以买入”的阈值直接下结论，但会复用以下维度：

1. 周线趋势是否还支持持有
2. 日线结构是否破坏
3. 60 分钟或尾盘是否出现转弱信号
4. 当前盈亏和原计划相比是否已达到止盈/止损/减仓条件
5. 市场情绪是否进入高潮或恐慌极值

### 周线趋势 25

重点看：

1. 周线价格是否在 `MA26/EMA26` 上方
2. 周线均线是否上行
3. 周线 MACD 是否支持多头
4. 是否接近明显周线压力位

### 日线回调质量 20

重点看：

1. 是否属于系统内回调，而不是追高
2. 是否回踩 `MA26`、平台支撑或零轴附近
3. 回调时是否缩量
4. 日线结构是否仍保持更高低点或至少未破坏关键支撑

### 尾盘触发质量 15

尾盘买入不是“尾盘抄底”，而是“尾盘确认没有走坏”。

优先加分项：

1. `14:30` 之后价格没有继续破位
2. 尾盘回到当日波动区间上半部
3. 尾盘量能改善，但不是失控抢筹
4. 60 分钟结构转强或至少不再转弱
5. 收盘前仍站在日线关键支撑上方

减分项：

1. 尾盘继续创新低
2. 跌破日线支撑后没有收回
3. 仅因消息刺激拉升，缺少结构支撑

### 板块强度 10

至少说明：

1. 所属板块是否同步走强
2. 个股是否弱于板块
3. 是否存在板块高潮末端风险

### 收益风险比 15

必须能说明：

1. 计划买入价
2. 结构止损价
3. 目标价或通道上轨
4. 预估收益风险比

若无法给出止损价或收益风险比，直接判定不可交易。

### 执行纪律与市场情绪 15

市场情绪先做定性，再映射到执行分：

1. `积极但未过热`：可正常评估
2. `分歧震荡`：可评估，但提高标准
3. `极度亢奋`：原则上不追，不因尾盘拉升放松标准
4. `极度恐慌`：原则上不抄底，除非结构清晰且用户明确要求模拟

执行纪律还要检查：

1. 是否已到允许时段
2. 是否已完成买前清单
3. 是否存在计划外冲动交易
4. 是否因回本、消息、情绪而入场

## 正式选股输出格式

正式选股或模拟选股时，输出保持这个结构：

1. 当前模式：`正式选股` 或 `模拟选股`
2. 当前时间与是否满足 `14:30` 规则
3. 候选池交易日：`T-1` / `当日`
4. 是否已完成 `T` 日尾盘复核
5. 市场情绪判断：乐观 / 中性 / 谨慎 / 极值回避
6. 数据来源：用户提供、仓库文件、外部机器摘要
7. 候选股列表

每只股票至少给出：

1. 代码和名称
2. 总分
3. 六项分数拆解
4. 评分基准日：`T-1盘后预评分` 或 `T日尾盘重评分`
5. 结论：`推荐` / `观察` / `不交易`
6. 触发原因
7. 主要风险
8. 若可行则补充计划买入价、止损价、目标区

若时间未到 `14:30`，结论最高只能写 `观察`。
若输入仅为 `T-1` 候选池且没有 `T` 日尾盘复核信息，结论最高只能写 `观察`。

## 交易日志格式

默认写入文件：`records/trade_journal.jsonl`

每条记录尽量包含：

```json
{
  "timestamp": "2026-06-15T14:52:00+08:00",
  "event_type": "buy",
  "mode": "formal",
  "symbol": "000001",
  "name": "示例股票",
  "pool": "日线回调池",
  "score_total": 84,
  "score_breakdown": {
    "weekly_trend": 22,
    "daily_pullback": 16,
    "close_trigger": 13,
    "sector_strength": 8,
    "reward_risk": 13,
    "discipline_emotion": 12
  },
  "market_emotion": "分歧震荡",
  "plan": {
    "buy_price": 12.34,
    "stop_price": 11.86,
    "target_price": 14.2,
    "position": "10%"
  },
  "execution": {
    "actual_price": 12.36,
    "shares": 1000,
    "followed_plan": true
  },
  "notes": "周线多头+日线回调+尾盘未破位",
  "missing_fields": []
}
```

如果用户只提供一句自然语言，也要尽量结构化后写入。

## 日志追加规则

1. 使用本地文件编辑方式追加，不重写整份历史。
2. 同一笔交易的买入和卖出是两条记录。
3. 若卖出时能计算结果，补充 `pnl_amount`、`pnl_pct`、`buy_score`、`sell_score`、`trade_grade`。
4. 若无法计算，不要编造，保留空字段或写入 `missing_fields`。

## 卖出分析输出格式

卖出分析时，输出保持这个结构：

1. 当前模式：`卖出分析`
2. 是否匹配到买入日志
3. 持仓摘要：买入日、买入价、当前价、浮盈亏、原止损价、原目标区
4. 当前结构判断：周线 / 日线 / 盘中或尾盘
5. 市场情绪判断
6. 结论：`继续持有` / `减仓` / `卖出` / `仅观察`
7. 触发依据
8. 主要风险
9. 若建议动作，说明更偏向 `止损`、`止盈`、`结构失败`、`纪律修正` 中哪一类

卖出分析优先检查这些条件：

1. 是否触发原止损
2. 是否达到原目标价或通道上轨
3. 是否出现周线趋势反转或日线结构失败
4. 是否放量滞涨、长上影、顶背离、连续创新高失败
5. 是否属于计划外买入后的纪律修正

若用户要求“现在能不能卖”，但没有当前价格、当前走势或图表描述，必须明确说信息不足，不能给高置信度建议。

## 回答风格

1. 先给结论，再给分数和理由。
2. 不要把低置信度推断说成事实。
3. 只要缺少关键价格，就明确说“不能形成可执行买入建议”。
4. 涉及外部机器选股结果时，标注为 `外部机器分析，仅供辅助`。
